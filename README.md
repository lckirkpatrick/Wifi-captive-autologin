# WiFi Captive Portal Auto-Login (Android)

## Purpose
Automate acceptance of captive Wi-Fi portals on Android using a profile-driven,
accessibility-based engine.

## Core Principles
- Profile-specific per SSID
- Modular architecture
- Accessibility-only interaction
- Safe automation (timeouts, cooldowns)

## Branch Strategy
- dev: auto-approved PRs
- stage: auto-approved after smoke tests
- main: manual approval
